# Decision Brief Format

Present decisions using this structure:

## 1. BLUF (Bottom Line Up Front)

Your recommendation in one sentence. Don't bury the lead.

## 2. Problem Statement

What problem you're solving. Lets user verify alignment.

## 3. Why User Input Is Needed

- If only one right answer: say so, explain why, proceed
- If tradeoffs exist: explain the core tension

## 4. Opportunity

Why this matters. What we gain by getting it right.

## 5. Options with Pros/Cons

For 2 options:

**Option A: [Name]**
- Pros: [benefits]
- Cons: [drawbacks]

**Option B: [Name]**
- Pros: [benefits]
- Cons: [drawbacks]

For 3+ options, use a table:

| Option | Description | Pros | Cons |
|--------|-------------|------|------|
| A | ... | ... | ... |
| B | ... | ... | ... |
| C | ... | ... | ... |

## 6. Evaluation & Recommendation

Synthesize the tradeoffs and explain your recommendation.

## 7. Next Step

Clear action requested from user.

---

## Guidelines

- **Concise**: Every word earns its place
- **Scannable**: User gets gist in 10 seconds
- **Honest**: Don't hide downsides
- **Actionable**: Clear question or next step
- **No time estimates**: Focus on what, not how long
